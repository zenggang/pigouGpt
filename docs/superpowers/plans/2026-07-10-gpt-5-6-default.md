# GPT-5.6 Default Model Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GPT-5.6 to Pigou AI Console, send `gpt-5.6-sol` upstream, and make it the one-time upgraded default for every user while retaining GPT-5.5 and GPT-5.4 choices.

**Architecture:** Centralize the Next.js application's supported model IDs and default in `src/lib/types.ts`, consume them from both the client and server request validation, and keep the separately deployed ECS worker's validation synchronized. Use a versioned browser settings key to migrate every existing user once while preserving reasoning effort, and explicitly persist the new default when creating conversations so existing database column defaults cannot override application behavior.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Node.js verification scripts, MySQL schema, ECS Node.js worker.

### Task 2: Centralize and apply the GPT-5.6 model configuration

**Files:**
- Modify: `src/lib/types.ts`
- Modify: `src/lib/sub2api.ts`
- Modify: `src/lib/conversations.ts`
- Modify: `src/components/ConsoleApp.tsx`

- [ ] **Step 1: Define the supported models and default once**

Replace the standalone `PigouModel` union in `src/lib/types.ts` with:

```ts
export const PIGOU_MODELS = ["gpt-5.6-sol", "gpt-5.5", "gpt-5.4"] as const;
export type PigouModel = (typeof PIGOU_MODELS)[number];
export const DEFAULT_PIGOU_MODEL: PigouModel = "gpt-5.6-sol";
```

- [ ] **Step 2: Reuse the shared allowlist in request validation**

Import `PIGOU_MODELS` as a value in `src/lib/sub2api.ts`, remove the local `MODEL_ALLOWLIST`, validate using `PIGOU_MODELS.includes(input.model)`, and change the error to:

```ts
throw new Error("当前支持 GPT-5.6、GPT-5.5 和 GPT-5.4。");
```

- [ ] **Step 3: Explicitly persist the new default for new conversations**

Import `DEFAULT_PIGOU_MODEL` in `src/lib/conversations.ts`, then change `createConversation` to insert the model explicitly:

```ts
await execute(
  `insert into conversations (id, user_id, title, model)
   values (?, ?, ?, ?)`,
  [id, userId, "新的会话", DEFAULT_PIGOU_MODEL],
);
```

Return `model: DEFAULT_PIGOU_MODEL`.

- [ ] **Step 4: Use the shared model type and default in the client**

Import `DEFAULT_PIGOU_MODEL`, `PIGOU_MODELS`, and `PigouModel` from `@/lib/types`, replace the local union with `type Model = PigouModel`, and define labels:

```ts
const MODEL_LABELS: Record<Model, string> = {
  "gpt-5.6-sol": "GPT-5.6",
  "gpt-5.5": "GPT-5.5",
  "gpt-5.4": "GPT-5.4",
};
```

Use `DEFAULT_PIGOU_MODEL` for React state and settings fallback. Render `MODEL_LABELS[item]` while keeping each option value equal to its exact model ID.

- [ ] **Step 5: Implement the one-time browser settings migration**

Use these keys:

```ts
const SETTINGS_KEY = "pigou-ai-console-settings-v2";
const LEGACY_SETTINGS_KEY = "pigou-ai-console-settings-v1";
```

When V2 is absent but V1 exists, parse V1, create a migrated object with `model: DEFAULT_PIGOU_MODEL` and the old valid reasoning effort (otherwise `medium`), write V2, remove V1, and return the migrated object. Add a focused `//` comment explaining that ignoring the old stored model is required to apply the product-wide GPT-5.6 default exactly once. For valid V2 settings, preserve any supported manually selected model.

- [ ] **Step 6: Keep cached historical conversations readable**

Update `isConversationSummary` to accept models via:

```ts
typeof record.model === "string" &&
PIGOU_MODELS.includes(record.model as Model)
```

This allows cached summaries using GPT-5.6 while retaining older GPT-5.5 and GPT-5.4 records.
